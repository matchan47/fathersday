## Commit Rule
- コミットは最低単位でのコミットとしてください。途中の状態や複数単位をまとめてコミットするのは基本NGです。
- コミットメッセージは第三者が見ても意味のわかるようにしてください。具体的な内容を記載してください。（「諸々対応」などはNGです）
- コミットがどのような内容なのかひと目で分かるように、コミットには頭に識別子を付与してください。
例：  
[識別子] コミットメッセージ
### 識別子の種類
- fix：バグの修正
- add：新規機能追加・ファイルの追加
- update：バグではない機能修正
- remove：削除（ファイル）
例：  
[fix] リサイズした際にスライダーが動かなくなる不具合修正  
[add] 標準ボタンのコンポーネント作成  
[update] ボタンのマウスオンアニメーションの秒数を修正  
[remove] 商品用ダミー画像の削除
